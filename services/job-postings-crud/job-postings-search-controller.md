# Контроллер поиска вакансий

<!-- markdownlint-disable MD060 -->
Работает с PostgresSQL:

- База данных `joposcragent`
- Схема `job_postings`
- Таблица `postings`

## Получение списка вакансий по простому отбору

`GET /job-postings/list`

| Входной параметр       | Источник       | Описание |
|------------------------|----------------|----------|
| `{uuid}`               | query-параметр | Внутренний UUID вакансии |
| `{uid}`                | query-параметр | Внешний uid вакансии |
| `{title}`              | query-параметр | Подстрока названия вакансии |
| `{company}`            | query-параметр | Подстрока названия компании |
| `{evaluationStatus}*`  | query-параметр | Повторяемый; литерал enum или `NULL` → см. ниже |
| `{responseStatus}*`    | query-параметр | Повторяемый; литерал enum или `NULL` → см. ниже |
| `{page}`               | query-параметр | Номер страницы (по умолчанию `1`) |
| `{size}`               | query-параметр | Размер страницы (по умолчанию `20`) |

(\*) Для `evaluationStatus` и `responseStatus`:

- допускается несколько значений в query (например `evaluationStatus=RELEVANT&evaluationStatus=NULL`);
- каждое значение, кроме регистронезависимого литерала **`NULL`**, должно быть известным литералом соответствующего enum; иначе **HTTP 400**;
- строка **`NULL`** добавляет к условию по колонке вариант **`IS NULL`**;
- несколько не-NULL значений образуют **`IN (...)`**;
- комбинация **`IN`** и **`NULL`** объединяется по **ИЛИ** для этой колонки.

Колонка `response_status` в БД **NOT NULL**, значение по умолчанию **`NEW`**.

Алгоритм работы:

1. Формирует запрос к таблице `postings`, добавляя условия по переданным параметрам (условия по разным полям объединяются по **AND**):
   1. Если передан `{uuid}`, добавляет условие `uuid` = `{uuid}`
   2. Если передан `{uid}`, добавляет условие `uid` = `{uid}`
   3. Если передан `{title}`, добавляет условие `title LIKE '%{title}%'`
   4. Если передан `{company}`, добавляет условие `company LIKE '%{company}%'`
   5. Если передан хотя бы один `{evaluationStatus}`, добавляет фильтр по `evaluation_status` согласно правилам `NULL` / `IN` выше
   6. Если передан хотя бы один `{responseStatus}`, добавляет фильтр по `response_status` согласно правилам `NULL` / `IN` выше
2. Сортирует строки по `uuid` по возрастанию
3. Применяет пагинацию: смещение `({page} - 1) * {size}` строк, лимит `{size}` строк
4. Возвращает `HTTP 200` с объектом `JobPostingsList`: массив `list` и число `totalPages` (правило то же, что в OpenAPI)
5. При возникновении любого не перехваченного исключения возвращает `HTTP 500` с текстом исключения в теле ответа

## Получение вакансий по списку UUID

`POST /job-postings/search-query/by-uuids`

| Входной параметр | Источник     | Описание                           |
|------------------|--------------|------------------------------------|
| 📌 `{uuids}`     | тело запроса | Объект `UuidsList` со списком UUID |

Алгоритм работы:

1. Получает список UUID из поля `list` тела запроса
   1. Если массив `list` пустой, возвращает `HTTP 400` с текстом "Массив list не может быть пустым"
2. Выбирает из таблицы `postings` все строки, у которых `uuid` входит в полученный список
3. Если не найдено ни одной строки, возвращает `HTTP 404`
4. Возвращает объект `JobPostingsList`, используя найденные строки
5. При возникновении любого не перехваченного исключения возвращает `HTTP 500` с текстом исключения в теле ответа

## Получение списка новых вакансий (отсутствующих в БД)

`POST /job-postings/search-query/non-existent`

| Входной параметр | Источник     | Описание                                            |
|------------------|--------------|-----------------------------------------------------|
| 📌 `{uids}`      | тело запроса | Объект `JobPostingsUidsList` со списком внешних uid |

Алгоритм работы:

1. Получает список uid из поля `list` тела запроса
   1. Если массив `list` пустой, возвращает `HTTP 400` с текстом "Массив list не может быть пустым"
2. Выбирает из таблицы `postings` все строки, у которых `uid` входит в полученный список
3. Из исходного списка `{uids.list}` исключает те uid, для которых в шаге 2 была найдена строка в БД
4. Если в результате не осталось ни одного uid (все уже присутствуют в БД), возвращает `HTTP 404`
5. Возвращает объект `JobPostingsUidsList` с оставшимися (новыми) uid
6. При возникновении любого не перехваченного исключения возвращает `HTTP 500` с текстом исключения в теле ответа
