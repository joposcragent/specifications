# Контроллер настройки порогов релевантности

Работает с PostgresSQL:

- База данных `joposcragent`
- Схема `settings`
- Таблицы `relevance_thresholds`

## Получить все пороги релевантности

`GET /relevance-thresholds/list`

Алгоритм работы:

Алгоритм работы:

1. Выбирает все записи из таблицы `relevance_thresholds`
   1. Если записей нет, возвращает `HTTP 404`
2. Возвращает объект `RelevanceThresholdsList`, используя данные таблицы
3. При возникновении любого не перехваченного исключения возвращает `HTTP 500` с текстом исключения в теле ответа

## Получить порог релевантности конкретного типа

`GET /relevance-thresholds/{type}`

| Входной параметр | Источник      | Описание                 |
|------------------|---------------|--------------------------|
| 📌`{type}`       | path-параметр | Тип порога релевантности |

Алгоритм работы:

1. Ищет строку в таблице `relevance_thresholds`, у которой `type` = `{type}`
   1. Если нет такой строки, возвращает `HTTP 404`
2. Возвращает объект `RelevanceThresholdItem`, используя поля найденной строки
3. При возникновении любого не перехваченного исключения возвращает `HTTP 500` с текстом исключения в теле ответа

## Установить порог релевантности конкретного типа

`POST /relevance-thresholds/{type}`

| Входной параметр | Источник      | Описание                 |
|------------------|---------------|--------------------------|
| 📌`{type}`       | path-параметр | Тип порога релевантности |
| 📌`{body}`       | тело запроса  | json-объект              |

Алгоритм работы:

1. Находит или создает строку таблицы `relevance_thresholds`, у которой `type` = `{type}`
2. Заполняет поля строки:
   1. `value` = `{body}.value`
   2. `updated_at` = `now()`, если строка была найдена; если строка была создана, то `updated_at` = `null`
3. При успешной записи в БД возвращает `HTTP 200`
4. При возникновении любого не перехваченного исключения возвращает `HTTP 500` с текстом исключения в теле ответа
