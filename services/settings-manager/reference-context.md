# Контроллер управления эталонным контекстом

Работает с PostgresSQL:

- База данных `joposcragent`
- Схема `settings`
- Таблицы `reference_context`

## Получение эталонного контекста

`GET /reference-context`

Алгоритм работы:

1. Если таблица `reference_context` пустая, то есть не содержит строк, возвращает `HTTP 404`
2. Получает из таблицы `reference_context` первую (или просто произвольную одну) строку
3. Возвращает объект `ReferenceContext` используя поля найденной строки
4. При возникновении любого не перехваченного исключения возвращает `HTTP 500` с текстом исключения в теле ответа

## Установка эталонного контекста

`POST /reference-context`

| Входной параметр | Источник     | Описание                   |
|------------------|--------------|----------------------------|
| 📌 `{context}`   | тело запроса | Текст эталонного контекста |

Алгоритм работы:

1. Получает вектор эталонного контекста запросом [sentence-transformer/text/vectorize]
   1. Запрос `POST http://localhost:8000/sentence-transformer/text/vectorize`
   2. В поле `text` тела запроса передает `{context}`
   3. Результат успешной обработки запроса обозначим `{context_vector}`
2. Получает из таблицы `reference_context` первую (или просто произвольную одну) строку
   1. Если таблица пустая, то добавляет одной строку и использует ее
3. Заполняет поля полученной строки:
   1. `text` = `{context}`
   2. `vector` = `{context_vector}`
   3. Если строка была найдена, а не создана, то `updated_at` = `now()`. Если стока была создана, что`updated_at` = `null`
4. При успешной записи в БД возвращает `HTTP 200`
5. При возникновении любого не перехваченного исключения возвращает `HTTP 500` с текстом исключения в теле ответа

[sentence-transformer/text/vectorize]: ../sentence-transformer/readme.md#вычислить-векторное-представление-текста
