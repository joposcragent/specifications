# Контроллер настройки CSS-селекторов

Работает с PostgresSQL:

- База данных `joposcragent`
- Схема `settings`
- Таблицы `css_query_selectors`

## Получение списка CSS-селекторов

`GET /query-selector/list`

Алгоритм работы:

1. Выбирает все записи из таблицы `css_query_selectors`
   1. Если записей нет, возвращает `HTTP 404`
2. Возвращает объект `SearchQueriesList`, используя данные таблицы
3. При возникновении любого не перехваченного исключения возвращает `HTTP 500` с текстом исключения в теле ответа

## Получение селектора

`GET /query-selector/{selectorType}`

| Входной параметр    | Источник      | Описание      |
|---------------------|---------------|---------------|
| 📌 `{selectorType}` | path-параметр | Тип селектора |

Алгоритм работы:

1. Ищет строку в таблице `css_query_selectors`, у которой `type` = `{selectorType}`
   1. Если нет такой строки, возвращает `HTTP 404`
2. Возвращает объект `CssQuerySelectorItem`, используя поля найденной строки
3. При возникновении любого не перехваченного исключения возвращает `HTTP 500` с текстом исключения в теле ответа

## Установка селектора

`POST /query-selector/{selectorType}`

| Входной параметр    | Источник      | Описание                        |
|---------------------|---------------|---------------------------------|
| 📌 `{selectorType}` | path-параметр | Тип селектора                   |
| 📌 `{selector}`     | тело запроса  | Строка, содержащая CSS-селектор |

Алгоритм работы:

1. Ищет строку в таблице `css_query_selectors`, у которой `type` = `{selectorType}`
   1. Если нет такой строки, возвращает `HTTP 404`
2. Обновляет следующие поля найденной строки в БД:
   1. `selector` = `{selector}`
   2. `updated_at` = `now()`
3. При успешной записи в БД возвращает `HTTP 200`
4. При возникновении любого не перехваченного исключения возвращает `HTTP 500` с текстом исключения в теле ответа

## Очистка селектора

`DELETE /query-selector/{selectorType}`

| Входной параметр    | Источник      | Описание      |
|---------------------|---------------|---------------|
| 📌 `{selectorType}` | path-параметр | Тип селектора |

Алгоритм работы:

1. Ищет строку в таблице `css_query_selectors`, у которой `type` = `{selectorType}`
   1. Если нет такой строки, возвращает `HTTP 404`
2. Удаляет найденную строку из БД
3. При успешной записи в БД возвращает `HTTP 200`
4. При возникновении любого не перехваченного исключения возвращает `HTTP 500` с текстом исключения в теле ответа
