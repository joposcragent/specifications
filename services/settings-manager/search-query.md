# Контроллер управления поисковыми запросами

Работает с PostgresSQL:

- База данных `joposcragent`
- Схема `settings`
- Таблицы `search_queries`

## Получение списка настроенных поисковых запросов

`GET /search-query/list`

Алгоритм работы:

1. Выбирает из таблицы `search_queries` все строки
2. Возвращает объект `SearchQueriesList`, используя поля таблицы
3. При возникновении любого не перехваченного исключения возвращает `HTTP 500` с текстом исключения в теле ответа

## Добавление нового поискового запроса

`POST /search-query/{entityUuid}`

| Входной параметр  | Источник      | Описание                |
|-------------------|---------------|-------------------------|
| 📌 `{entityUuid}` | path-параметр | UUID поискового запроса |
| `{queryString}`   | тело запроса  | Текстовая строка        |

Алгоритм работы:

1. Проверяет наличие в таблице `search_queries` строки с `uuid`=`{entityUuid}`
   1. Если строка найдена, возвращает `HTTP 409`
2. Добавляет строку в `search_queries`, заполняя поля следующим образом:
   1. `uuid` = `{entityUuid}`
   2. `query` = `{queryString}`
3. При успешной фиксации изменений в БД возвращает `HTTP 200`
4. При возникновении любого не перехваченного исключения возвращает `HTTP 500` с текстом исключения в теле ответа

## Обновление поискового запроса

`PATCH /search-query/{entityUuid}`

| Входной параметр  | Источник      | Описание                |
|-------------------|---------------|-------------------------|
| 📌 `{entityUuid}` | path-параметр | UUID поискового запроса |
| `{queryString}`   | тело запроса  | Текстовая строка        |

Алгоритм работы:

1. Проверяет наличие в таблице `search_queries` строки с `uuid`=`{entityUuid}`
   1. Если строка отсутствует, возвращает `HTTP 404`
2. Обновляет следующие поля найденной строки
   1. `query` = `{queryString}`
   2. `updated_at` = `now()`
3. При успешной фиксации изменений в БД возвращает `HTTP 200`
4. При возникновении любого не перехваченного исключения возвращает `HTTP 500` с текстом исключения в теле ответа

|-------------------|---------------|-------------------------|
| 📌 `{entityUuid}` | path-параметр | UUID поискового запроса |

## Получение поискового запроса

`GET /search-query/{entityUuid}`

| Входной параметр  | Источник      | Описание                |
|-------------------|---------------|-------------------------|
| 📌 `{entityUuid}` | path-параметр | UUID поискового запроса |

Алгоритм работы:

1. Выбирает из таблицы `search_queries` строку, у которой `uuid` = `{entityUuid}`
   1. Если такой строки нет, возвращает `HTTP 404`
2. Возвращает объект `SearchQueriesItem`, используя данные найденной строки
3. При возникновении любого не перехваченного исключения возвращает `HTTP 500` с текстом исключения в теле ответа
