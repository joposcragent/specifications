# job-postings-evaluator

Сервис автоматического оценивания вакансий.

Работает с PostgreSQL:

- База данных `joposcragent`
- Схема `job_postings`
- Таблица `postings`

---

## `POST /evaluate/sync/list`

| Входной параметр | Источник                       | Описание                                             |
| ---------------- | ------------------------------ | ---------------------------------------------------- |
| 📌 `{list}`       | тело запроса, JSON-поле `list` | Список внутренних UUID вакансий (`UuidsListRequest`) |

Алгоритм работы:

1. Принимает тело запроса и извлекает из него список идентификаторов вакансий.
2. Проверяет корректность запроса; при ошибке проверки или чтения тела формирует ответ с кодом состояния «неверный запрос» без тела.
3. Загружает параметры оценивания: порог релевантности для содержимого вакансии и опорный вектор контекста; при отсутствии или недопустимости параметров формирует ответ с кодом состояния «внутренняя ошибка» и текстовым телом с описанием проблемы.
4. Выбирает из хранилища вакансии, у которых идентификатор входит в переданный список и статус оценки — «новая» или «ожидает»; упорядочивает выборку по идентификатору.
5. Если подходящих вакансий нет, формирует ответ с кодом состояния «не найдено» без тела.
6. Для каждой выбранной вакансии по порядку следования в выборке выполняет действия [`POST /evaluate/sync/{jobPostingUuid}`](#post-evaluateasyncjobpostinguuid)
7. Формирует список результатов (идентификатор вакансии и итоговый статус оценки) и помещает его в тело ответа с кодом состояния «успех».
8. При иных неперехваченных сбоях формирует ответ с кодом состояния «внутренняя ошибка» и текстовым телом с трассировкой исключения.

## `POST /evaluate/sync/{jobPostingUuid}`

| Входной параметр      | Источник      | Описание                 |
| --------------------- | ------------- | ------------------------ |
| 📌 `{jobPostingUuid}` | path-параметр | Внутренний UUID вакансии |

Алгоритм работы:

1. Принимает идентификатор вакансии из пути запроса.
2. Проверяет корректность запроса; при ошибке формирует ответ с кодом состояния «неверный запрос» без тела.
3. Загружает параметры оценивания (порог релевантности для содержимого, опорный вектор контекста); при отсутствии или недопустимости формирует ответ с кодом состояния «внутренняя ошибка» и текстовым телом с описанием проблемы.
4. Выбирает из хранилища вакансии с данным идентификатором и статусом оценки «новая» или «ожидает»; упорядочивает по идентификатору (фактически не более одной записи).
5. Если подходящих вакансий нет, формирует ответ с кодом состояния «не найдено» без тела.
6. Для единственной выбранной вакансии:
   1. Определяет вектор содержимого (сохранённый или полученный по тексту содержимого).
   2. Вычисляет меру близости к опорному вектору контекста.
   3. Сравнивает меру с порогом и присваивает статус «релевантна» или «нерелевантна».
   4. Сохраняет статус оценки, релевантность, вектор содержимого и время обновления в отдельной транзакции.
7. Формирует тело ответа с одним полем — итоговым статусом оценки — и отдаёт ответ с кодом состояния «успех».
8. При иных неперехваченных сбоях формирует ответ с кодом состояния «внутренняя ошибка» и текстовым телом с трассировкой исключения.

## `POST /evaluate/async/list`

| Входной параметр | Источник          | Описание                                             |
| ---------------- | ----------------- | ---------------------------------------------------- |
| 📌 `{list}`       | тело, поле `list` | Список внутренних UUID вакансий, как в синхронном list |

Алгоритм работы:

1. Принимает тело запроса со списком идентификаторов вакансий.
2. Проверяет корректность запроса; при ошибке формирует ответ с кодом состояния «неверный запрос» без тела.
3. Запускает в фоне алгоритм [`POST /evaluate/sync/list`](#post-evaluatesynclist) и возвращает HTTP 200
4. При необработанном исключении в фоновой обработке пишет ошибку в журнал (обработчик необработанных исключений асинхронного выполнения).

## `POST /evaluate/async/{jobPostingUuid}`

| Входной параметр      | Источник | Описание                 |
| --------------------- | -------- | ------------------------ |
| 📌 `{jobPostingUuid}` | path     | Внутренний UUID вакансии |

Алгоритм работы:

1. Принимает идентификатор вакансии из пути запроса.
2. Запускает в фоне действия [`POST /evaluate/sync/{jobPostingUuid}`](#post-evaluateasyncjobpostinguuid).
3. Возвращает HTTP 200
4. При необработанном исключении в фоновой обработке пишет ошибку в журнал (обработчик необработанных исключений асинхронного выполнения).

## `POST /evaluate/async/batch`

| Входной параметр | Источник  | Описание                                                    |
| ---------------- | --------- | ----------------------------------------------------------- |
| 📌 `size`         | тело JSON | Целое число ≥ 1 (`BatchAsyncProcessingParametersRequest`) |

Алгоритм работы:

1. Принимает тело запроса с положительным целым числом `size` (не меньше единицы).
2. Проверяет корректность запроса; при ошибке формирует ответ с кодом состояния «неверный запрос» без тела.
3. Ставит в очередь фоновую задачу пакетной оценки и сразу формирует ответ с кодом состояния «успех» без тела.
4. В фоновой обработке пытается загрузить параметры оценивания; при невозможности загрузки или неполных данных прекращает обработку и пишет предупреждение в журнал.
5. Иначе выбирает из хранилища не более `size` вакансий со статусами «новая» или «ожидает», упорядоченных по идентификатору; при отсутствии записей пишет предупреждение в журнал и завершает обработку.
6. Иначе для каждой вакансии из пакета выполняет действия по алгоритму [`POST /evaluate/sync/{jobPostingUuid}`](#post-evaluateasyncjobpostinguuid).
7. Для каждой успешно обработанной вакансии пишет в журнал информационную запись с идентификатором вакансии и присвоенным статусом оценки.
8. При необработанном исключении в фоновой обработке пишет ошибку в журнал.
