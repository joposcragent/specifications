# Оркестратор (Flower)

## Назначение и маршрут

- **Раздел меню:** Главная → **Оркестратор** (внутренний маршрут, пункт виден, если задан `VITE_FLOWER_BASE_URL`).
- **Маршрут (пример):** `/orchestrator` — в контентной области оболочки открывается экран с **встроенным iframe** (URL Flower из env), без перехода на другую вкладку браузера.

## Макет (wireframe)

```text
  Главная
    Дашборд          -> внутренний роут
    Оркестратор      -> внутренний роут, iframe на VITE_FLOWER_BASE_URL
```

## Данные и API

- URL встраивания: **`VITE_FLOWER_BASE_URL`** (полный базовый URL Flower, тот же, что раньше открывали в новой вкладке).
- Сам SPA **не** проксирует и **не** дублирует API Flower: загрузка идёт в iframe напрямую с указанного origin. Если Flower запрещает встраивание (например `X-Frame-Options`), настройте сервер Flower / обратный прокси для разрешения frame-ancestor под ваш origin.

## Алгоритмы активных элементов

### Пункт меню «Оркестратор»

- **Клик:** `router` ведёт на `/orchestrator`; контент — iframe с `src` = `import.meta.env.VITE_FLOWER_BASE_URL`.
- Если env пустой, пункт в меню **не** показывается (как и ранее).

### Экран оркестратора

- **Показ:** заголовок и на всю ширину контентной колонки — iframe с фиксированной минимальной высотой (визуально «вытянутый» блок под рабочую область).
- **Пустой env:** предупреждение в UI, iframe не рендерится.

## Открытие страницы (mount)

1. Прочитать `VITE_FLOWER_BASE_URL` из сборки; при непустом значении задать `src` у iframe.

## Обновление страницы (F5)

1. URL iframe восстанавливается из той же env-константы в бандле.
