# Оркестратор (Flower)

## Назначение и маршрут

- **Раздел меню:** Главная → **Оркестратор**.
- **В приложении отдельного экрана нет:** пункт меню ведёт на внешний веб-интерфейс Flower.

## Макет (wireframe)

Внутри SPA отдельный контентный макет **не** отображается. В меню — одна строка наравне с «Дашборд».

```text
  Главная
    Дашборд          -> внутренний роут
    Оркестратор      -> внешняя ссылка
```

## Данные и API

- URL берётся из **`VITE_FLOWER_BASE_URL`** (полный базовый URL Flower).
- Запросы с фронта к Flower **не** обязательны: достаточно HTML-ссылки `<a href="..." target="_blank" rel="noopener noreferrer">`.

## Алгоритмы активных элементов

### Пункт меню «Оркестратор»

- **Клик:** открыть `import.meta.env.VITE_FLOWER_BASE_URL` в **новой вкладке**.
- **Не** показывать предупреждения вида «небезопасный localhost» и **не** добавлять обходных проверок URL: значение задаётся доверенной конфигурацией окружения.
- **Не** внедрять дополнительную бизнес-логику (редиректы, прокси) на фронте, кроме чтения env.

## Открытие страницы (mount)

Не применимо для внутреннего роутера. При первом рендере shell пункт меню просто доступен для клика.

## Обновление страницы (F5)

Не применимо к контенту SPA; ссылка в меню по-прежнему указывает на актуальный `VITE_FLOWER_BASE_URL` из сборки.
