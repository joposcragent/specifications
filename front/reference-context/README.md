# Эталонный контекст

## Назначение и маршрут

- **Раздел меню:** Настройки → **Эталонный контекст** (страница по умолчанию при входе в «Настройки»).
- **Маршрут (пример):** `/settings/reference-context`.

## Макет (wireframe)

```text
+------------------------------------------------------------------+
|  Заголовок «Эталонный контекст»                                   |
+------------------------------------------------------------------+
|  Многострочное поле (редактируемый текст эталона):                |
|  высота по умолчанию ~30% от области страницы, вертикальный      |
|  resize, прокрутка текста внутри поля                             |
+------------------------------------------------------------------+
|  Дата: updatedAt, иначе createdAt (read-only)                    |
+------------------------------------------------------------------+
|  Вектор эталона (read-only): моноширинный блок, прокрутка         |
+------------------------------------------------------------------+
|  [ Кнопка «Записать» / Save ]                                     |
+------------------------------------------------------------------+
```

## Данные и API

Базовый URL: `VITE_SETTINGS_MANAGER_BASE_URL`. Контракт: [services/settings-manager/openapi.yaml](../../services/settings-manager/openapi.yaml).

### Загрузка при открытии

- **Метод:** `GET /reference-context`
- **200:** тело `ReferenceContext`: `context` (строка), `vector` (массив чисел), `createdAt`, `updatedAt`
- **202:** эталонный контекст **не задан** — показать пустое поле ввода и пустой/нулевой вектор по согласованному UX; даты при отсутствии данных — по полям ответа или прочерки
- **500:** ошибка — сообщение пользователю, без маскировки под «пусто»

### Сохранение

- **Метод:** `POST /reference-context`
- **Content-Type:** `text/plain`, тело — текст эталонного контекста
- **200:** тело `ReferenceContextPersisted`: `vector`, `createdAt`, `updatedAt` (**без** поля `context` в контракте успешного ответа)

После успешного POST:

- Обновить на форме блок **вектора** и поля **дат** из ответа.
- Текст в поле ввода оставить тем, что пользователь отправил (или при желании UX повторно запросить `GET` — не требуется постановкой, если ответ не содержит `context`).

## Алгоритмы активных элементов

### Многострочное поле ввода

- Редактирование свободного текста; валидация формата на этой странице не требуется, кроме ограничений бэкенда при ошибке сохранения.
- Область редактирования: по умолчанию занимает ориентировочно **30% высоты** доступной области страницы (контейнер с явной высотой относительно окна); допускается **вертикальное изменение размера** пользователем (resize); при переполнении — **вертикальная прокрутка** содержимого внутри поля.

### Кнопка «Записать»

- **Клик:** отправить `POST /reference-context` с телом `text/plain` = текущее значение поля.
- **Во время запроса:** заблокировать кнопку и/или показать индикатор.
- **Успех 200:** подставить `vector`, `createdAt`, `updatedAt` из JSON ответа в read-only области.
- **Ошибка:** показать сообщение (snackbar / alert M3); поле ввода не очищать.

### Read-only «Обновлён / создан»

- Источник: **`updatedAt`**, если заполнено; иначе **`createdAt`** из последнего успешного `GET` или из ответа `POST`.

### Read-only «Вектор»

- Отобразить **полностью** все компоненты массива (моноширинный текст, переносы или одна строка со скроллом — на усмотрение UX); обязательна прокрутка при большой размерности.

## Открытие страницы (mount)

1. Выполнить `GET /reference-context`.
2. При **200** — заполнить поле `context`, блок вектора, даты.
3. При **202** — пустое состояние ввода и вектора по правилам выше.
4. При ошибке — уведомление, форма в безопасном состоянии (пусто или последнее известное — зафиксировать в реализации).

## Обновление страницы (F5)

Повторить полный `GET`; несохранённые правки в поле ввода теряются (стандартное поведение браузера), при необходимости в будущем можно добавить предупреждение о несохранённых данных — вне текущей постановки.
