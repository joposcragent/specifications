# Оболочка приложения (app shell)

## Назначение и маршрут

Общий каркас SPA: не отдельный маршрут, а layout, внутри которого рендерятся дочерние страницы (`RouterView`). Задаёт **вертикальный split** (меню / контент), ширину колонок, боковое меню, область контента и глобальные контролы (тема).

## Макет (wireframe)

```text
+------------------------------------------------------------------+
| [Nav] |·| [Content]                                    [≡] [T]  |
|       | |  <RouterView />  (резиновая ширина)                    |
|  ...  | |                                                      |
+------------------------------------------------------------------+
```

- **Левая колонка (Nav)**: навигация; ширина **задаётся пользователем** (в пределах min/max) и **сохраняется** в `localStorage`. Между Nav и контентом — **разделитель** для перетаскивания (изменение ширины).
- **Скрытие Nav**: кнопка в шапке контентной части **скрывает** левую колонку (и разделитель); **показ** — той же кнопкой. Состояние «свёрнуто» **сохраняется** в `localStorage`. При скрытии и при смене ширины **основная колонка и содержимое** заполняют доступную ширину (`min-width: 0`, flex, без фиксированного max у контентной обёртки).
- **Правая колонка (Content)**: `RouterView` в контейнере **на всю ширину** окна под шапкой (без жёсткого `max-width` у корневой обёртки контента).
- **Шапка контентной колонки**: слева — кнопка «меню / свернуть боковую панель», справа — переключатель **светлая / тёмная** тема.
- Пункты меню первого уровня — **увеличенный** кегль; вложенные — **меньший** кегль.

### Параметры по умолчанию (реализация)

| Параметр | Значение |
|----------|----------|
| Ширина Nav при первом визите | ~260px (диапазон ~200–480px) |
| Ключ `localStorage` ширины | `joposcragent-nav-width` |
| Ключ `localStorage` скрытия меню | `joposcragent-nav-collapsed` (`1` = скрыто) |

## Данные и API

Оболочка **не** вызывает бизнес-API сама по себе. HTTP-клиенты дочерних экранов используют базовые URL из переменных окружения (см. [корневой README](../README.md)).

- Axios (или эквивалент) настраивается с `baseURL` = значение соответствующей `VITE_*_BASE_URL` для каждого бэкенда.
- Вариант с reverse-proxy и same-origin описан в корневом README; CORS при прямых cross-origin URL — на стороне API.

## Алгоритмы активных элементов

### Иерархическое меню

- **Клик по родительскому разделу** (опционально): раскрыть/свернуть дочерние пункты, если реализовано аккордеоном; иначе родитель может быть только подписью без навигации.
- **Клик по дочернему пункту**: `router.push` на маршрут экрана; активный пункт подсвечивается по `route.name` / префиксу пути.
- **Пункт «Оркестратор»**: не внутренний роут — см. [orchestrator-flower/README.md](../orchestrator-flower/README.md).

### Изменение ширины меню

- **Перетаскивание вертикального разделителя** между Nav и контентом: изменить ширину Nav в пределах min/max, сохранить в `localStorage`.
- **Указатель**: `col-resize`; на время перетаскивания отключается выделение текста в документе.

### Скрытие / показ меню

- **Клик по кнопке в шапке контента**: переключить видимость Nav и разделителя; записать флаг в `localStorage`.

### Переключатель темы

- **Клик**: переключить тему приложения (светлая ↔ тёмная) в согласовании с выбранной M3-библиотекой (например Vuetify `useTheme`).
- **При смене**: записать выбор в `localStorage` (стабильный ключ, например `joposcragent-theme`), чтобы при следующем визите тема восстановилась.

### Область контента

- Рендер дочернего маршрута без дополнительной логики.
- **Ширина**: контентная область и внутренние блоки **адаптивны** по ширине viewport (таблицы и прочие элементы ужимаются в рамках колонки за счёт flex и при необходимости горизонтальной прокрутки у компонентов данных).

## Открытие страницы (первый заход / полная загрузка приложения)

1. Прочитать `localStorage` по ключу темы; если значение валидно — применить тему до первого paint (по возможности), иначе тема по умолчанию (светлая или системная — зафиксировать в реализации единообразно).
2. Прочитать сохранённую ширину Nav и флаг скрытия меню; применить к layout.
3. Отобразить layout с меню (или без, если свёрнуто) и областью контента.
4. Роутер активирует целевой дочерний маршрут (например `/` → дашборд); логика загрузки данных — в спецификации этого экрана.

## Обновление страницы (F5, повторный вход по URL)

1. Повторить чтение темы и параметров layout из `localStorage` (состояние Pinia сбрасывается).
2. Восстановить маршрут из URL; меню отразит активный пункт.
3. Дочерний экран сам выполняет свои запросы при `mount` (см. соответствующие `README.md`).
