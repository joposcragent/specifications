# Дашборд

## Назначение и маршрут

- **Раздел меню:** Главная → **Дашборд** (контентная страница по умолчанию для «Главная»).
- **Маршрут (пример):** `/` или `/home/dashboard` — зафиксировать в реализации роутера единообразно с остальными экранами.

## Макет (wireframe)

```text
+------------------------------------------------------------------+
|  Заголовок «Дашборд» (или в составе контентной шапки shell)      |
+------------------------------------------------------------------+
|  [Фильтры по title/company — если поддержаны бэкендом]           |
|  [Опционально: клиентские поля фильтра по колонкам без API]      |
+------------------------------------------------------------------+
|  Таблица вакансий (data table M3)                                 |
|  колонки: название | URL | превью текста | ... | пагинация        |
+------------------------------------------------------------------+
|  [Модальное окно: полный текст вакансии] — при открытии из ячейки |
+------------------------------------------------------------------+
```

## Данные и API

Базовый URL: `VITE_JOB_POSTINGS_CRUD_BASE_URL`. Контракт: [OpenAPI job-postings-crud][job-postings-crud-openapi].

### Список для таблицы

- **Метод:** `GET /job-postings/list`
- **Пагинация (серверная):** query `page`, `size`. Значения `< 1` на бэкенде нормализуются к `1`.
- **Ответ:** объект `JobPostingsList`: массив `list`, поле **`totalPages`** (целое, ≥ 0). Поля **`total`** в OpenAPI нет: при необходимости показа «всего записей» вычислять из `totalPages` и текущего `size` или договориться о расширении API отдельно.
- **Сортировка на сервере:** фиксирована контрактом — по `uuid` по возрастанию; **любая сортировка по колонкам в UI — только клиентская** (на текущей странице `list`).

### Фильтрация на бэкенде (использовать только где есть параметры)

| Query-параметр | Поведение |
|----------------|-----------|
| `evaluationStatus` | Один или несколько повторов → логика `IN` по enum |
| `responseStatus` | Один или несколько повторов → логика `IN` по enum `ResponseStatus` |
| `includeUnsetResponseStatus` | Булево, по умолчанию `false`. Если `true`, в выборку дополнительно попадают строки с `response_status IS NULL` (в сочетании с `responseStatus` условие по статусам и «без статуса» объединяется через **ИЛИ**) |
| `title` | Подстрока (`LIKE %...%`) |
| `company` | Подстрока |
| `uuid`, `uid` | Точечный отбор |

### Требуемый набор данных для дашборда (контракт)

На дашборде отображать только вакансии **в рассмотрении** в смысле продукта:

- `evaluationStatus === RELEVANT` (в запросе: повторяемый query `evaluationStatus=RELEVANT`);
- и (`responseStatus === NEW` **или** в БД `response_status` ещё не задан — `NULL`).

**Пример запроса списка:** `GET /job-postings/list?evaluationStatus=RELEVANT&responseStatus=NEW&includeUnsetResponseStatus=true&page=1&size=30` (плюс при необходимости `title`, `company`).

Контракт зафиксирован в [OpenAPI job-postings-crud][job-postings-crud-openapi] (версия **1.3.0** и выше).

### Колонки таблицы

| Колонка | Источник / поведение |
|---------|----------------------|
| Название | `title`, текст |
| URL | `url`, ссылка `target="_blank"`, `rel="noopener noreferrer"` |
| Текст вакансии | Первые **100** символов `content`; клик → модалка с полным `content` |
| Статус оценки | Dropdown enum `EvaluationStatus`; сохранение по завершении выбора |
| Статус рассмотрения | Dropdown enum `ResponseStatus` |
| Релевантность | Число `relevance` или «—», если поля нет / null |
| Компания | `company` |
| Дата публикации | `publicationDate` **как строка с бэка** — показывать как есть, без парсинга в Date для отображения |
| Дата загрузки | `createdAt` (или согласованное поле из DTO) |

### Инлайн-смена статусов

- **Оценка:** `POST /job-postings/{jobPostingUuid}/evaluation-status/{status}`
- **Рассмотрение:** `POST /job-postings/{jobPostingUuid}/response-status/{status}`

Успех: **2xx**, тело пустое — **не** перезапрашивать строку и страницу; локальное состояние строки = отправленное значение.

Ошибка: сеть, **4xx**, **5xx** — вернуть dropdown к **предыдущему** значению; показать сообщение об ошибке (snackbar или иной паттерн M3-библиотеки; «тултип» — если библиотека не даёт длительного тултипа на ошибку, использовать snackbar).

## Алгоритмы активных элементов

### Комбобокс размера страницы

- Допустимые значения: **30**, **50**, **80**, **130**.
- **По умолчанию:** 30.
- **При смене:** сбросить `page` на 1 (или оставить политику «текущая страница в пределах totalPages» — предпочтительно сброс на 1); выполнить `GET /job-postings/list` с новым `size`.

### Пагинация (серверная)

- Элементы управления страницами опираются на `totalPages` и текущий `page`.
- **При смене страницы:** новый запрос списка с теми же фильтрами и `size`.

### Фильтры

- **Поддержанные query на бэке** (`title`, `company`, `evaluationStatus`, …): при изменении — debounce по UX (опционально) → запрос с `page=1`.
- **По колонкам без бэка:** клиентская подстрока по уже загруженной странице `list` (не путать с серверной выборкой всей БД).

### Сортировка по заголовкам колонок

- Только **клиентская** сортировка видимых строк текущей страницы (данные уже получены).

### Ячейка превью `content`

- **Клик:** открыть модальное окно с полным текстом `content`, закрытие по кнопке / клику вне / ESC — по стандарту библиотеки.

### Dropdown статусов

- **Начало редактирования:** запомнить предыдущее значение в состоянии строки.
- **Выбор нового значения:** отправить соответствующий `POST`; при успехе обновить только локальное поле; при ошибке — откат и уведомление (см. выше).

## Открытие страницы (mount)

1. Установить UI в состояние загрузки (скелетон / индикатор в таблице).
2. Выполнить `GET /job-postings/list` с параметрами дашборда (`evaluationStatus=RELEVANT`, `responseStatus=NEW`, `includeUnsetResponseStatus=true`), `page=1`, `size=30` (или восстановить из query/session — опционально).
3. **Успех 200:** если `list` пуст — показать пустое состояние («нет подходящих вакансий»), **не** как ошибку.
4. **Ошибка сети / 5xx / 4xx:** показать сообщение об ошибке; таблица без данных или с сохранённым последним успешным состоянием — на выбор UX, но пустой успешный ответ не смешивать с ошибкой.

## Обновление страницы (F5, повторный вход на маршрут)

1. Сбросить клиентскую сортировку/локальные фильтры без бэка (если не сохраняются в URL).
2. Повторить сценарий mount: запрос списка с дефолтными или восстановленными из URL параметрами.
